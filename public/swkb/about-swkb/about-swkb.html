The SoftWare Knowledge base is a collection of chunks of information created and maintained in association with the x-square project of NodeJS-Houston.

The main goal of the project is to learn the tools and processes of developing a modern data-driven web site.

Secondarily, we may create some useful components and/or a working web site.

In order to capture a concise abstract  of the information learned in order to recall it later and to be useful to others, this is a condensed form of the uncontrolled and minimally organized information available on the web.

This will work in conjunction with project-specific information managed in parallel, and linked, but separate.

swkb will be organized by generally well identified tools, components that become a part of x-square or any similar project.  Each wil get it's own subfolder. This will maintain modular, organized information.

A second type of tool I call collaboration tools, such as slack and trello.  They are used in the process of the team collaborating, but not necessarily as final components.  They will also have a similar separate subfolder under swkb/

Certain types of pages may be indicated by keywords in their pagenames. For example a page with summaries and links to tutorials may include tut- in the page name.  Usually a tool involves ten or twenty key concepts. These may get pages each, with prefix hbt-  for HandleBarsTerms eg templates, precompile, expression, route,...)

The objective in folder structure and page-naming is to make it humanly easy to
    1) add new information so it will be accessible and not duplicated - you can see what is there.
         similarly know which folder a given chunk should be placed in.
    2) keep the number of files in a folder to about 20 or max 50.
          This makes scanning the filenames to create links or to find info manageable
     3) to identify holes of potentially useful chunks missing to add them.
     4) Keeping chunks to be visible in one screen without scrolling, unless good reason.
